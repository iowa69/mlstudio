Welcome to MLSTudio

A guided 4-step workflow gets you from a folder of assemblies to an interactive minimum spanning tree.

  1. 1
    Pick an input folder. Use the sidebar's 1 · Input folder panel → click Browse… and select a folder of .fasta assemblies (optionally with paired _R1/_R2.fastq.gz reads).
  2. 2
    Pick a scheme. The 2 · Scheme dropdown shows everything cached locally. To grab a new one, use the Scheme catalog panel or run mlstudio schemes pull-eskapee.
  3. 3
    Set options. Defaults are sensible: auto-tuned identity/coverage, all cores, no AMR. Toggle the AMR scan on if you want resistance genes alongside typing.
  4. 4
    (Optional) Metadata CSV. First column = sample name; every other column becomes a colorable field on the tree.
  5. ▶
    Click Analyze. Progress streams over WebSocket. When the run finishes you'll be flipped to the MST tab automatically.

What you get

  • 🌳 MST — interactive minimum spanning tree, cluster halos, drag-to-pin nodes, live colour-by.
  • 📊 Table — per-sample ST + per-locus allele matrix, cluster-tinted rows, full-hits TSV export.
  • 💊 AMR — gene-by-sample matrix from AMRFinderPlus, never contributing to the MST distance.
  • 📈 Statistics — call-quality histograms, cluster size distribution, per-locus call rate.
